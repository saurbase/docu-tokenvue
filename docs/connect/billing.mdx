# Billing

Billing in TokenVue shows the current workspace plan, token usage, payment status, and billing history.

At the moment, TokenVue runs on the **Free Basic** operating model. This means all platform features are enabled, token quota is unlimited, no payment method is required, and no billing history is generated.

![Billing Overview](/img/docs/bl-1.png)

## Current Plan

The active plan is **Free Basic**.

This plan includes:

- All platform features
- Unlimited token quota
- No monthly charge
- No token overage
- No payment card requirement
- Community support

## Billing Summary

The Billing page shows key workspace billing details, including:

- Active plan
- Upcoming bill
- Token usage
- Payment card status

For the Free Basic plan, the upcoming bill is always `$0`.

## Token Quota

TokenVue tracks workspace token usage from virtual key request logs.

The quota for Free Basic is currently unlimited, so usage is shown for visibility only.

![Token Quota](/img/docs/bl-3.png)

The token quota section shows:

- Tokens consumed
- Request count
- Monthly limit
- Overage status

## Subscription Management

The Subscription Management section displays available plans.

Free Basic is currently selected. Paid plans may be visible for comparison, but they are disabled while TokenVue is using the free operating model.

| Plan | Status |
| --- | --- |
| Free Basic | Active |
| Pro | Reserved for future billing |
| Team | Reserved for future billing |

![Subscription Management](/img/docs/bl-2.png)

## Payment Methods

No payment method is required for the Free Basic plan.

The payment method section remains empty while billing is disabled.

## Billing History

Billing history shows invoices and receipts when paid billing is active.

For the Free Basic operating model, there are no invoices, receipts, or scheduled billing records.

## In Short

Billing gives teams visibility into plan status and token usage.

In the current TokenVue setup, billing is free, quota is unlimited, paid plans are disabled, and no payment method is required.
